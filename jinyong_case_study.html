Jin-Yong's Novels Themed-bookstore

「 Project Goals 」

上面有D引言, 落下小小, 先講Project Goals

通過建立金庸小說主題書店，而不只是電子閱讀器應用程序，提出更有吸引力的體驗。 旨在提供與用戶相關的方法，為滿足他們在小說和觀眾之間閱讀的愛好帶來樂趣。

這是一個基於我自己的經驗的個人激情項目。 雖然我不在數字出版領域，但我確實喜歡重新設計金庸小說電子閱讀器的想法。 我承擔了設計產品 UX 和 UI 所涉及的所有角色。

「 Research 」

項目開始時，我做了一個應用程序分析和了解市場，包括可用性和功能。 然後，為了驗證這些觀察結果，我需要進行一些用戶訪談和使用電子閱讀器的經驗。 並嘗試了解他們的想法。 我使用 Facebook 小組投票對 8 個人（閱讀數字版和印刷版）進行了有關該應用程序的採訪。 另外，我分配了一些簡單的任務來看看他們個人如何使用它並閱讀行為模式。

任務 1：導航整個應用程序
任務 2：閱讀不同場景：家庭/公共交通
任務 3：時間花費
任務 4：檢查閱讀狀態和歷史記錄

接下來，我通過親和力映射來嘗試通過用戶引用進一步對問題進行分組。 發現了一些痛點：

「 Pain Point 痛點」

第1個:

不完整的電子閱讀器功能

他們都提到，沒有提供可追踪的小說和歷史，給他們的閱讀和歷史造成了不便，尤其是長期閱讀的用戶，並希望評估時間。 2/8 也提到了一些功能錯誤或崩潰。

第2個: 讀取裝置

讀取裝置

在手機屏幕上閱讀不舒服是大多數評論用戶。 他們提到需要經常在屏幕上長時間滾動和滑動。 有 5/8 的人說在 MTR 閱讀有乾擾和肩痛。 1 人表示在亞馬遜 Kindle 上閱讀體驗比手機好。

第3個:

缺乏書籍類型

4/8 的人表示沒有他們感興趣的金庸小說相關書籍。 最後他們在谷歌上搜索併購買了印刷版。

第4個:

醜陋的界面

5/8 的人表示該應用程序沒有吸引力，尤其是狂熱的讀者； 他們說“無聊，感覺讀文章”； “它不適合我”。

「 同情用戶面對」

第1點:

每個讀者都有自己的個性化閱讀習慣。它只是書籤頁面是不夠的。沒有時間跟踪功能會導致最令人沮喪的事情，也會降低他們的閱讀效率和動力。因此，他們需要可追溯的信息，例如歷史當前閱讀內容以及預期閱讀時間，尤其是導致應用程序崩潰和多任務受眾。

第2點: 讀取裝置

長時間在手機屏幕上會導致他們不舒服和眼睛疲勞的問題。狂熱的讀者非常“享受”小說，平均花費 3.5 小時以上。因此，他們尋找更好的設備和場所來更流暢地閱讀。因此，他們最終傾向於印刷版或亞馬遜 Kindle 版。這就是為什麼用戶更喜歡實體書而不是電子書的原因。

第3點:

他們喜歡在看過小說後發現小說相關的書籍和武術信息。但沒有提供相關的選擇。因此，他們最終會購買其他電子書店或實體版。這是失去潛在觀眾的機會。

第4點:

整體觀感無法讓用戶感興趣。金庸小說是武俠傳奇。一些狂熱的讀者表達了他們對他作品的喜愛。缺乏小說元素和章節插圖讓我注意到他們很失望。

「 Persona 角色 」

編譯這些數據後，我設置了 3 個角色，最後得出了 2 種角色。 它們的目標和閱讀行為在某種程度上是不同的，這讓我可以知道他們實際需要關注的地方：

Primary Persona 主要角色:

MAX作為中產階級忙碌的媒體作家，小說愛好者，享受家庭時光和舒適的家，需要方便的設備和愉快的環境，因為他不斷需要寫作技巧和想法，他可以隨時參考， 讓他在漫長的一天工作後能更好地享受閱讀時光。

Secondary Persona 次要角色：

Andy 是一名會計和活躍的遊戲愛好者，他喜歡守時，希望一份盡可能輕鬆舒適的工作。 他渴望工作和愛好的生活平衡。 不常閱讀，喜歡金庸小說的意象。 他關心按時閱讀，這樣他就可以把時間花在生活中的事情和各種愛好上。

「 Competitive Analysis 競爭分析 」

通過對市場上競爭對手的快速測試，這給了我當前應用程序模式的清單。 還給了我製作設計說明的可能方式的想法。

「 IDEATION & SOLUTION 構思與解決方案 」

「 Problem Statement 」

金庸博士，華語世界偉大的小說家。 他的 15 部作品肩負著將人與小說聯繫起來的宏偉使命，深刻影響了過去幾十年香港流行文化的發展。 提供可靠的閱讀體驗是讓用戶輕鬆探索小說的最大挑戰之一。

“如何更好地傳達金庸小說電子書閱讀器的價值，讓讀者在舒適和愉悅的同時體驗金庸小說？”

開發主題書店

在啟發我創建主題書店的發現階段。考慮到這一點，我決定將電子書閱讀器應用程序改造為金庸主題書店，通過重構整個體驗並獲得價值。在用戶研究階段的結果中，我發現主題書店的方法與同時實現的用戶目標非常吻合。這種方法為讀者提供了多層次的意象和感受，以增強閱讀體驗並滿足狂熱的讀者需求。

豐富的書籍類型和增值內容

通過在晉學（金學）和小說研究書籍中增加書籍類型來擴大目標讀者。此外，通過創作以金庸小說為主要元素的知名人物和武俠的互動增值內容，與目標受眾的期望產生共鳴。

豐富主題書店氛圍

通過創建集成到 UI 中的數字插圖，為用戶提供了幻想和遊戲化。這將改善金庸主題書店的氛圍，同時滿足實體印刷用戶群。

增強應用內閱讀器的實用性

閱讀狀態跟踪是他們在用戶研究期間想要的功能。 這是幫助用戶滿足他們的閱讀行為或意外原因停止閱讀的關鍵。 研究表明，能夠跟踪進度是動力的重要來源。 只看到頁碼可能會令人沮喪。 通過顯示易於理解的百分比而不是頁碼。 此外，評估一本需要時間閱讀的小說也可以減輕用戶壓力。 讓它即使是最厚的小說也更友好； 這尤其是為了實現想要有效管理時間和多任務用戶的目標。

另外，受訪者提到，長時間閱讀大多感覺眼睛疲勞。 所以，我決定找到適合閱讀的配色方案，如何減少正文的視覺壓力和最佳的易讀性。

閱讀平台策略

合適的設備是他們正在尋找的關鍵之一。 根據用戶的實際情況，他們傾向於將在家看小說作為大多數場景和動機。 我打算使用“平板電腦優先”的方法進行設計，其中橫向屏幕幾乎與物理硬拷貝大小相同。 因此，讓他們對閱讀感到舒適並獲得最佳效用至關重要。 平板秤屏幕不僅可以降低棄用率和眼睛健康問題，還可以專心閱讀。 以下是支持“平板優先”方法的精選數據：

其作品《天龍八部》的大部分文字 :
+1.5 million

平均花在小說上的時間：
1h 40m

平均完成小說的時間：
15 days

平均讀取次數：
2 - 3次

在用戶研究期間。 他們需要在閱讀過程中頻繁地向下滑動或向左滑動，這會降低用戶使用該應用程序的動力。 手機雖然使用廣泛，但不適合長時間閱讀，尤其是文字量大的小說。 如果我們長時間在手機屏幕上閱讀，這會使我們的眼睛工作得更辛苦一些。

「 ITERATION 強調任務流程和反饋 」

之後我做左 Paper Mockup流程圖，可以做初步ITERATION，3個重要process：#1：Home； #2：Create own card； #3：Scan card process。 我同2名參與者做左Testing，有助我了解Users, 同App交互interacting過程： 睇下使用上情况, 同埋個完成度

1. Home Interaction 方面

係呢度，我盡量考慮番現實場景，UI上保持單手操作流暢。 Design左垂直滾動方法 數字卡片盒，User feedback, 巨地既感覺 “幾似真實” “主要功能容易睇到”。 提高了Users使用的動力和預期。 根據參與者的觀察，將 Card list 突出Digital card卡片，我刪除了中間Filter 圖標既描述text，剩係Show icon。

2. Create own card 方面

新流程能夠輕鬆引導完成任務, 我觀察到, 參與者最常用可定制選項, 而不是內置模板，雖然順利完成任務, 。 雖然順利完成任務, 參與者說完成步驟四步有D長。 所以，我將步驟, 從四步簡化為三步, 同埋Fix 1d交互元素問題。

3. Scan card 方面

Testing時, User擔心會 Miss data Capture唔到, 所以, 係Final flow，相應地添加了卡片Preview。 如果卡片數據Capture唔正確，卡片Preview可以幫巨地進行編輯。

1 Home

主頁作為 “數字卡片盒” Design, 係App既核心Screen，右上角Head icon係User profile 同App setting, 中間係Card list, 顯示當前卡片狀態, 所有核心功能係底部, 中間係Exchange交換卡 Button, 左邊係 Scan卡Button, 右邊係 Search聯繫人Button。 Card list 垂直Scroll, 貼近真實揾卡片動作, 比User感覺“真實”的卡片盒。

By default, 最新卡片係Card list頂部。 中間有 Filter panel, 目的係比User快速Sort out聯繫人list，例如最近的聯繫人、 收藏夾、 公司等等。 另外, 長按卡片, 可以編輯、 刪除卡 同Share卡。

Click卡片入去聯繫人詳細信息。 User可以直接向左滑動電話號碼, 直接打撥打電話。 另外，Users可以 Add 同巨地有關係既人。 呢個功能係inpire ITERATION階段。 比User可以跟進巨地既個人網絡。

2 Create own card

如原型所示，當User click頭像，入去“我的卡片盒”，呢個係自己卡片的中心。 可以 Create自己的卡片，有3種模式：

1. 重新開始 - Step by Step 去create卡片
2. 企業 - 創建後從雲存儲訪問
3. Scan卡 in-App OCR scanner capture實體卡數據

咁個Demo係 Start all over

第 1 步 - Import step中，Users可以從 Address book 導入，email 或者 vCard format通過他們的手機同步，以便聯繫信息始終是最新的，可以防止手動數據輸入錯誤。 另外, 如果需要，Users可以 Add 頭像。

入左基本資料後, 係第2步 有2個選項, 關於卡片Visual：

- 自定 Customizable
User可以使用工具、裁剪和顏色疊加 自定義視覺元素

- 用 Build-in template
User可以用Template，系統將加載導入的數據，有助於簡化整個過程

咁個Demo 第 3 步 係自定Customizable - 呢度可以比Users自已調整卡片佈局、 Font 同埋卡上信息顯示 ON/OFF。 Users調整時, 會睇到即時變化。 卡片設置有更靈活的選項。

另外, Enable “設置為企業模板”，Cloud base，可以同團隊共享。 Card design 同埋Layout Startard 左做1個Template, 避免浪費時間重複工作。

******呢度回應番User Pain point, 整個過程簡化左, 可以好好降低左實體卡用戶門檻, 重視視覺上同引導操作。 (1.1 注重Step by Step互動)

3 Exchange Parnter card

Users create左第一張 Card後。 通過NFC 功能, 交換過程幾乎完全自動化。 如果對方有呢個rollCase App，咁就更方便。

咁如Demo所示，Home page中間有Exchange交換 button, 向上Swipe 個button, 咁User 可以揀邊張卡去換。 然後，靠近對方既Mobile，過程都係自動完成，同埋有Success動畫, 完成任務。 另外, 可以用 QR 碼（創建卡片時自動生成）或者 vCard format 同其他人Share卡片，目的係 for 冇呢個 App 既人。

交換後，Home page 會顯示新卡。 然後，長按卡片, 可以編輯、 刪除卡。 仲可以經由電子郵件、 WhatsApp、 Facbook 和 Skype 等去Share。

4 Scan card

如您所見，Click “掃描” Button開 Camera，然後，影一張實體卡片，in-App OCR 功能可即時 Detect到, 卡片上每個字段，然後, 從卡片中Capture數據, 然後頂部顯卡片Preview。 自動Fill入去填姓名、 職務、 公司名稱等信息, 同時係可編輯的格式，如果有任何不正確的資料，User可以睇番頂部Preview圖, 手動修復。

Import後，如果Users唔需要看到卡，可以Click “保留相機拍攝” 選項, 是保留拍攝shot作為卡外觀。 目的係, 針對實體卡Users, 可以立即完成任務, 有助簡化整個過程。

******呢度消除都係番User Pain point, (1.3 消除從實體卡到數字化的手動工作) 有彈性可以唔洗做Design, 都可以有唔錯既卡外觀。
