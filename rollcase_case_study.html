「 Project Goals 」

呢個係一個Digital卡片APP, 第1段係引言, 落下小小, 先講Project Goals

重新創造一個數字卡片平台，以用戶為中心, 更有效率既體驗。 同時通過簡化流程, 減底紙卡用戶數字化的差距，

仲有1個比較高既願景, 係無紙化，全世界每年生產 1000 億張卡片，其中 58% 最終被掉去垃圾箱，然後送去填田區。 (inpire 新聞) (Solo project 負責所有 UX 和 UI task)

Problem Discovery 「 問題發現 」

卡片仍然是一種商務禮儀和社會習俗。 我們暫時不太可能, 睇到卡片從商業環境中消失。

係早期研究時，很明顯, 今日仍然有好多人使用紙卡片。 每當佢地整理佢地既卡片時，有一D好殘, 唔updated, 就會掉去垃圾箱，呢個就會造成污染影響。 係目前卡片App體驗中，UUsers任務比較複雜，既阻礙了紙卡用戶走向數字化的動力，亦比左挫敗感現在用緊App users，亦意味著用戶將重新使用紙卡，呢個問題反映左數字卡片app, 無咁廣泛接受既原因 .

「 Research 」

首先去了解用戶, 理解目標先。 揾左 10 位使用紙卡片, 有用過數字卡片App的人, 係Facebook 、 同埋係whatsApp group 做左調查。 問左D題關於用卡片情況, 例如:

拿到名片時，您如何處理？

花多少時間在上面, 去整理佢地既卡片？

有没有過呢類App? 有咩評價? 等等

佢地都有自己的挫敗感 同使用方式。 下面係總結左佢地既問題：

“我經常把卡片放在西裝袋裡，有時我冇時間去整理, 最終會浪費時間揾卡片”

“開會後回到Office時整理卡片，大約 3-4 分鐘入去手機”

“最近有用過APP，但操作複雜, 沒有足夠的提示怎麼做”

“個App幾有用, 唔喜歡加新卡時既步驟, 最後uninstall左 ”

「 了解競爭對手 」

大多數競爭應用程序都有繁瑣的步驟，這可能是它不受歡迎的原因。 例如在 Haystack 的調查結果中：

任務不流暢, 沒有足夠的指導性, 每一步都需要停底一段時間來了解，並且冇關於如何做的指導

我完成了卡片創建步驟，至少需要 4 分鐘，而且單手操作也不合適。

核心任務button不是主屏幕的主要位置，放左係menu。

「 痛點並找出行為模式 」

下圖係sort out左Research問題, 歸納左3個痛點 :

第1個痛點 ：冇時間去組織卡片

兩種類型Users既共同痛點; 不會故意花時間和精力整理名片, 佢地覺得係額外的工作。 大部分受訪者表示仍然用紙卡, 另外, 佢地已經做生意很長時間，而且存儲了大量需要整理的卡片。

第2個痛點：User flow 複雜

令到紙卡Users造成了左門檻,感到沮喪, Users對複雜步驟 ，, 並且也反映了當前用戶無動機去使用, 付合唔到users實際需求, 佢地會重新用紙卡。

「 Insights 見解 」

消化左後, 有3點見解:

第1點: 目前, 佢地點樣整理自巳卡片？

事實上，大多數人收到卡片後, 冇即時去整理, 另外, 亦都未必額外會花時間, 手動將卡片資料入到手機, 或者放入自已卡片盒。 亦好容易忘記或有時會丟失。 如果要同某人聯繫, 他們通常會浪費時間找卡。

第2點: App比較少人用

做research時, 10個人只有2個人用, 但它們同時覺得App難用, 或者放棄, 通常，用戶只有簡單的方法, 或清晰App才會對App感興趣, 如果可用性方面做得好, 發現到潛在用戶, 有空間開發Digital Card。

第3點: 佢地不擔心環境污染的影響！

環境污染, 不單只係是用定唔用App, 對於紙卡用戶, 如果卡片上資料有變化，佢地無辦法即時製作名片，佢地必須重印一疊全新卡片。 佢地唔見左卡片, 或者重印卡片時，都會產生額外既污染。 換句話說，適合紙卡用戶更重要, 可以減少環境污染問題。

「 解決正確的問題 」

分析他們實際面臨的真正問題, 2 類用戶Point Of View Statements

“作為一個經常處理成堆名片的紙卡用戶，在使用應用程序時, 希望在沒有任何說明的情況下直觀地操作，因為要花時間評估每一張卡片, 並清除我並不真正需要的那些，這很麻煩。 "

“作為一個經常與客戶面對面的數字卡應用程序用戶，想要自動化, 並且在與應用程序交互時沒有任何技巧和顧慮，因為他們想要避免手動輸入錯誤, 並且感到不安，可以有助於消除額外工作和跟進, 呢個很重要。”

「 Problem Statement 」

“點為人設計一種無壓力、可以高效去管理卡片, 同時令實體卡片用戶參與數字化卡片App, 從而達到環保目的？”

「 假設陳述 」

“我們認為名片用戶的問題是，他們希望在收集名片時以一種易於管理的方式放置，雖然工作忙碌並感到麻煩，但準備繼續減少對環境的影響，會以高效的方式鼓勵他們 進一步管理名片習慣的轉變，以免浪費時間在名片整理上。”

「 IDEATION 構思 」

主要通過Step base流程, 更容易完成任務 <- 針對第2痛點：操作複雜

Task flow 係個App重要組成部分, 包含三個核心功能：卡片創建、卡片交換和紙卡數字化, 提供1個更視覺化的Step base flow，減輕Users認知負擔, 專注任務, 而不是學習1個新事物, 提高兩種users類型的動機, 適應現實生活中場景使用。

之後, 下面第1點 降低任務複雜度，更容易數字化

一個新功能, 可以將拍攝卡片後既圖像, 保留為數字卡片外觀, 可以跳過設計卡片的過程 - 適合想要快速完成任務, 或者唔知道點樣設計卡片視覺既users。 既可以讓紙卡用戶輕鬆地進行數字化，也可以加快任務時間。 <- 針對第2痛點：任務操作複雜,

每個Step 都有輕鬆既標題，例如 “選擇你喜歡的卡片teamplate?” 、 “我既信息是咩”, 標題唔係命令式Step1做呢樣, Step2做呢樣, 輕鬆引導進一步的行動決策。 消除users壓力。

- 提供Template feature，可以加快完成, 唔需要任何設計技巧, 亦都可加快流程

- 提供雲的存儲 - 可以重複使用卡片template，例如有一定規模公司, 可以標準化佢地卡片，避免浪費時間重複工作。

第2點 通過 NFC 交互功能, 轉變卡片既整理習慣 <- 針對第1痛點：冇時間去組織卡片, 感覺額外工作

減少了額外的卡片組織和事後時間

構建1個NFC 流程，卡信息交換即時和自動，避免數據輸入錯誤和更好的準確性, 透過呢個便利性, 希望可以激勵User使用呢個App, 同埋改變佢地卡片管理習慣, 減少事後的額外時間做管理工作和跟進。

新增1個Notes功能, 卡片交換後, 可以比Users即時set schedules 同 Reminder, 例如Drop低開會後跟進事情, 避免用額外App。

第3點 1個數字“名片盒”方式 提高易用性, 鼓勵User用呢個App

將Home page做成 Digital 卡片盒 既設計，好似現實生活中既卡片盒一樣。 比純粹的聯繫人列表, 容易理解, 視覺上會更好, 另外，將關鍵功能任務放係Home度, 容易D操作同執行。 同時著重單手操作，適應現實生活場景。

3 Digital Card Exchange

第3個demo展示番, NFC交換卡片操作......

Home page中間有Exchange 交換Button, 向上Swipe個Button, 就出現自己個card list, User 可以揀邊張卡去交換。 過程雙方都要enable NFC功能, 然後，靠近對方既手機，就會自動完成, 完成會有動畫顯示, 2個step完成。 亦都有 QR 碼（在創建卡片時提供選項）For 冇甪呢個App 交換。

交換後，就跳番去Home 會顯示新卡。 長按卡片可以編輯、 刪除、 Share 可以經由Email、 WhatsApp、 Facbook 和 Skype 去Share。

Card Detail page裡面, 可以 Drop 事情notes, 有時間選項, 同埋 Set Reminder。 可能約左下次開會跟進, 重要的事情, 唔洗用額外 App記低。 (幾配合到User開會後情境)

通過呢個NFC自動化交換過程。 激勵User去整理名片的習慣, 減少事後的額外工作和時間。

4 紙卡數字化

第4個demo展示番由紙卡轉做Digital操作.......

當Users收到紙卡片時，係Home Click “掃描” Button, 就會開左 Camera，然後，影一張實體卡片, OCR 功能 Detect到, 卡片上每個字段，然後, 從卡片中Capture數據, 然後自動Fill入去下一步填資料欄位, 姓名、 職務、 公司名稱等等信息, 同時係可編輯的格式，如果有任何不正確的資料，User可以睇番頂部Preview圖, 手動去改。

入左data後，如果Users唔需要卡片外觀，可以Click “保留相機拍攝” 選項, 保留之前影底左作為卡片外觀。 可以立即完成任務, 整個過程比較簡單快捷。

******呢度消除都係番Pain point 2 操作複雜, 消除不必要手動工作, 亦都有彈性, 唔洗做卡片Design, 簡化過程。 對有大量卡片轉做Digital比較方便
