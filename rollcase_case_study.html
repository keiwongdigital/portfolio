「 Project Goals 」

點解唔用自己address book? 因為實際上會收到卡片, 呢個App係希望達成無紙化, 當然要User有環保意識前題

上面有D引言, 落下小小, 先講Project Goals

重新設計一個數字化卡片平台，以用戶為中心, 為人創造1個無縫體驗。 同時通過簡化流程, 降低實體卡用戶的准入門檻，從而增加潛在市場。

仲有1個比較高既願景, 係無紙化，全世界每年生產 1000 億張名片，其中 58% 最終被扔進垃圾箱，然後填田區。

「 Research 」

首先要做既係, 通過研究去了解用戶, 理解目標先。 揾左 12 位每天使用實體卡, 數字卡應用程序的人, 係Facebook 、 同埋係whatsApp 做左投票。 佢地都有自己的挫敗感 同使用方式。 下面係總結左佢地既問題：

“開會後, 我經常把卡片放在西裝袋裡，有時我冇時間去整理”

“我最近使用應用程序，但我不喜歡入卡時的複雜步驟”

“紙卡大小有限，有時我會使用 Google Keep 或落紙筆記”

「 Synthesising Finding 」

下圖係User problem歸納左3個痛點 :

第1個痛點 ：冇時間去組織卡片, 感覺麻煩

管理卡片, 同時係數字卡或實體卡用戶既共同痛點。 佢地感到不耐煩, 覺得係額外的工作。 大部分受訪者表示仍然用實體卡片, 佢地已經做生意很長時間，而且存儲了大量需要整理的卡片。

第2個痛點：操作複雜

App可用性方面，體驗唔太好, Users對複雜步驟 感到沮喪，並且在執行任務時, 唔夠清析D指導, 亦都付合唔到實際需求。

第3個痛點：功能上不足

受訪者表示需要額外的應用程序使用，當前的App功能, 無法滿足他們的任務和期望。

「 Empathize User Facing 」

用同理心去消化左後, 有3點見解:

第1點: 佢地目前點樣管理卡片？

事實上，沒人有時間整理聯繫人, 並手動將卡片入到手機, 或者放入卡片盒。 是很容易忘記或有時會丟失。 Users總係希望節省時間。

第2點: 香港比較少人用

很明顯, 今日仍有大量人用實體卡。 呢度發現到潛在用戶, 所以有大量空間開發Card App。 另外, 係可用性方面過程感覺複雜, 激發唔到User使用動機, 亦增加左實體卡User門檻。

第3點: 佢地不擔心環境污染的影響！

每次佢地組織既卡片時，就會有一些卡片, 最終進入垃圾箱。 另外, 如果某人卡片上資料有變化，佢地無辦法即時製作名片，佢地必須重印一疊全新卡片。 另外, 佢地唔見左卡片, 或者重印卡片時，都會產生額外的污染。

「 Competitive Analysis 競爭分析 」

做左一些研究，看看目前 競爭對手之間 功能，揾個市場上定位, 發現到主要區別有以下：

Haystack - 優點：強烈視覺效果 / 缺點：工作流程太多；

Biz-Card - 優點：信息清析明顯 / 缺點：較難操作；

Business Card Holder - 優點：容易訪問 / 缺點：視覺效果不佳；

「 IDEATION 構思 」

之後, 咁我構思有3點解決方案:

第1係 重新構思流程, 更容易完成任務 <- 針對第2痛點：操作複雜 (可以降低實體卡用戶進入門襤)

- 重新構思1個更視覺化的Step by Step flow，可以輕鬆引導User進一步的行動決策。

- 提供現成既Template，唔需要任何設計技巧都可以完成

- 用輕鬆的標題, 例如 “你想點設計卡片?” 、 “我既信息是咩”, 而唔糸叫User Step1做呢樣, Step2做呢樣, 能夠減輕User既認知負擔, 挫敗感。

第2點 消除從實體卡到數字化的手動工作 <- 針對第2痛點：操作複雜 (可以降低實體卡用戶進入門襤)

通過App裡面既 OCR 掃描功能，可以自動Capture 卡片Data, 自動Fit入去, 同時係可編輯格式。 目的係，可以減少用戶對手動輸入錯誤的挫敗感。 幾乎完全自動化的操作

另外，如果用戶不需要Design, 可以將拍攝卡片後既圖像, 保留為數字卡片外觀, 亦都可加快流程。 減低實體卡User進入門檻, 幾乎完全自動化的操作。

為企業用戶提供基於雲的模板。user可以重複使用卡片template，避免浪費時間重複工作。

第3點 增加便利性, 去轉變整理卡片習慣 <- 針對第1痛點：冇時間去組織卡片, 感覺麻煩, 同第3個痛點：功能不足

通過 NFC 功能簡化交換流程； 第1.打開你想分享的卡片； 第2. 靠近enable NFC 的設備或（相同的應用程序）設備； 這很適合在開會後, 目的係, 激勵佢地改變卡片管理方式，從而減少事後既精力和時間。

增加左1個Reminder功能, 可以比Users為未來既事情, 卡片交換後, 可以比Users即時set Reminder, 有時間同alert功能, 例如drop低下次開會, 可以避免左, 額外App使用。

第4點 1個數字“名片盒”方式 提高易用性

將Home page做成 “Digital Cardcase” Design，好似現實生活中既Cardcase一樣。

另外，將關鍵功能任務放係Home page度, 可以保持user單手操作, 容易D執行。

「 IDENTITY 驗證 」

之後我做左 Paper Mockup 做初步驗證，揾左2名參與者做左Testing，有助我了解Users 使用上情况, 同埋個任務完成度。

1. Home Interaction 方面

係呢度，我考慮番Users現實使用場景，保持單手操作流暢。 Design左垂直滾動方法 數字卡片盒，User feedback, 佢地既感覺 “幾似真實” “主要功能容易睇到”。 提高了Users使用的動力和預期。 根據參與者觀察，將 Card list 突出Digital card卡片，我刪除了中間Filter 圖標既描述text，剩係Show icon。

2. Own card creation 方面

新流程能夠輕鬆引導完成任務, 我觀察到, 參與者最常用可定制卡片選項, 而唔係內置Template, 雖然順利完成任務, 但參與者說完成步驟四步有D長。 所以，我將步驟, 從四步簡化為三步, 同埋Fix左 1D交互元素問題。

3. Import physical card 方面

Testing時, User擔心會 Miss data Capture唔到, 所以, 係Final flow，相應地添加了卡片Preview。 如果卡片數據Capture唔正確，卡片Preview可以幫佢地進行編輯。

經過幾輪 Paper mockup testing A/B 測試，確定哪個對Users有效。 觀察佢地D用, 情緒 同習慣係最重要既。

1 Home

第1個demo展示番Home interation 同基本操作, 同想係 Card list 塭聯絡人打電話比巨。

主頁作為 “數字卡片盒” Design, 係App既核心Screen，右上角Head icon係User profile 同App setting, 中間係Card list, 顯示當前卡片狀態, 所有核心功能係底部, 中間係Exchange交換卡 Button, 左邊係 Scan卡Button, 右邊係 Search聯繫人Button。 Card list 垂直Scroll, 貼近真實揾卡片動作, 比User感覺“真實”的卡片盒。

By default, 最新卡片係Card list頂部。 中間有 Filter panel, 目的係比User快速Sort out聯繫人list，例如最近的聯繫人、 收藏夾、 公司等等。 另外, 長按卡片, 可以編輯、 刪除卡 同Share卡。

Click卡片入去聯繫人詳細信息。 User可以直接向左滑動電話號碼, 直接打撥打電話。 另外，Users可以 Add 同佢地有關係既人。 呢個功能係inpire ITERATION階段。 比User可以跟進佢地既個人網絡。

******目的係, (Solution point 4: Home page 提高了可訪問性 ) 整個Home page UI上處理, 提高易用性, 係1個數字名片盒 design, 可以好順手去操作, 去Contact 其他人。

2 Own card creation

第2個demo展示番Create自己的卡片操作.......

如原型所示，當User click頭像，入去“我的卡片盒”，呢個係自己卡片的中心。 可以 Create自己的卡片，有3種模式：

1. 重新開始 - Step by Step 去create卡片
2. 企業 - 創建後從雲存儲訪問
3. Scan卡 in-App OCR scanner capture實體卡數據

咁個Demo係 Start all over

第 1 步 - Import step中，Users可以從 Address book 導入，email 或者 vCard format通過他們的手機同步，以便聯繫信息始終是最新的，可以防止手動數據輸入錯誤。 另外, 如果需要，Users可以 Add 頭像。

入左基本資料後, 係第2步 有2個選項, 關於卡片Visual：

- 自定 Customizable
User可以使用工具、裁剪和顏色疊加 自定義視覺元素

- 用 Build-in template
User可以用Template，系統將加載導入的數據，有助於簡化整個過程

咁個Demo 第 3 步 係自定Customizable - 呢度可以比Users自已調整卡片佈局、 Font 同埋卡上信息顯示 ON/OFF。 Users調整時, 會睇到即時變化。 卡片設置有更靈活的選項。

另外, Enable “設置為企業模板”，Cloud base，可以同團隊共享。 Card design 同埋Layout Startard 左做1個Template, 避免浪費時間重複工作。

******呢度解決番 Pain point 操作複雜, 整個過程簡化左, 可以好好降低左實體卡用戶門檻, 重視視覺上元素同引導操作。 (1.1 注重Step by Step互動)

3 Exchange Parnter card

第3個demo展示番交換卡片操作.......

Users create左第一張 Card後。 通過NFC 功能, 交換過程幾乎完全自動化。 如果對方有呢個rollCase App，咁就更方便。

咁如Demo所示，Home page中間有Exchange 交換Button, 向上Swipe個Button, 咁User 可以揀邊張卡去換。 然後，靠近對方既Mobile，過程都係自動完成，同埋有Success動畫, 完成任務。 另外可以用 QR 碼(創建卡片時自動生成), 或者 vCard format 同其他人Share卡片，目的係for 冇呢個 App既人。

4 Import physical card

第4個demo展示番由實體卡轉做Digital卡片操作.......

如您所見，Click “掃描” Button開 Camera，然後，影一張實體卡片，in-App OCR 功能可即時 Detect到, 卡片上每個字段，然後, 從卡片中Capture數據, 然後頂部顯卡片Preview。 自動Fill入去填姓名、 職務、 公司名稱等信息, 同時係可編輯的格式，如果有任何不正確的資料，User可以睇番頂部Preview圖, 手動修復。

Import後，如果Users唔需要看到卡，可以Click “保留相機拍攝” 選項, 是保留拍攝shot作為卡外觀。 目的係, 針對實體卡Users, 可以立即完成任務, 有助簡化整個過程。

******呢度消除都係番User Pain point, (Solution point 2 消除從實體卡到數字化的手動工作) 有彈性可以唔洗做Design, 都可以有唔錯既卡外觀。
