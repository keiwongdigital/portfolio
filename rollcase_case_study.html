「 Project Goals 」

呢個係一個Digital卡片APP, 第1段係引言, 落下小小, 先講Project Goals

重新創造一個數字卡片平台，以用戶為中心, 更有效率嘅體驗。 同時通過簡化流程, 減底紙卡用戶數字化嘅差距，

呢個App, inspire 1段環保新聞, 有1個比較高嘅願景, 係無紙化，全世界每年生產 1000 億張卡片，其中 58% 最終被掉去垃圾箱，然後送去填田區。 (inpire 新聞) (Solo project 負責所有 UX 和 UI task)

「 Problem Discovery 而家存在咩問題 」

卡片仍然是一種商務禮儀和社會習俗。 我地暫時不太可能, 睇到卡片從商業環境中消失。

係早期研究時，很明顯, 今日仍然有好多人使用紙卡片。 每當佢地整理佢地嘅卡片時，有一D好殘, 唔updated, 就會掉去垃圾箱，呢個就會造成污染影響。 係目前卡片App體驗中，UUsers任務比較複雜，嘅阻礙了紙卡用戶走向數字化嘅動力，亦比左挫敗感現在用緊App users，亦都意味著用戶, 將會用番紙卡，呢個問題反映左數字卡片app, 無咁廣泛接受嘅原因 .

「 Research 研究 」

首先去了解用戶, 理解目標先。 揾左 10 位使用紙卡片, 有用過數字卡片App嘅人, 係Facebook 、 同埋係whatsApp group 做左調查。 問左D題關於用卡片情況, 例如:

拿到名片時，您如何處理？

花多少時間在上面, 去整理佢地嘅卡片？

有没有過呢類App? 有咩評價? 等等

佢地都有自己嘅挫敗感 同使用方式。 下面係總結左佢地嘅問題：

“我經常把卡片放在西裝袋裡，有時我冇時間去整理, 最終會浪費時間揾卡片”

“開會後回到Office時整理卡片，大約 3-4 分鐘入去手機”

“最近有用過APP，但操作複雜, 沒有足夠嘅提示怎麼做”

“個App幾有用, 但唔鐘意加新卡時嘅步驟, 最後uninstall左 ”

「 Finding out Pain Points & Analysis 識別痛點, 同分析 」

下圖係sort out左Research問題, 歸納左3個痛點 :

第1個痛點 ：冇時間去組織卡片

兩種類型Users嘅共同痛點; 不會故意花時間和精力整理名片, 佢地覺得不耐煩和缺乏時間。 大部分受訪者表示仍然用紙卡, 另外, 佢地已經做生意很長時間，而且存儲了大量需要整理嘅卡片, 最終都要浪費時間進行整理

第2個痛點：任務複雜

令到紙卡Users造成了左門檻, 感到挫敗感, 反映了當前用戶無動機去使用, 意味著付合唔到users實際需求, 亦反映佢地會, 重新用番紙卡。

「 Insights 見解 」

消化左後, 有3點見解:

第1點: 目前, 佢地點樣整理自巳卡片？

事實上，大多數人收到卡片後, 冇即時去整理, 另外, 亦都未必手動將卡片資料入到手機, 或者放入自已卡片盒。 亦好容易忘記或有時會丟失。 如果要同某人聯繫, 佢地通常會浪費時間找卡片。

第2點: App比較少人用

做research時, 10個人只有2個人用, 但它們同時覺得App難用, 導至放棄, 用戶只係鍾意簡單, 清晰嘅App, 至會感興趣, 其實有好多空間開發Digital Card App。

第3點: 佢地不擔心環境污染嘅影響！

環境污染係全人類嘅問題, 唔單只係是用定唔用App, 每當佢地整理佢地嘅卡片時，唔updated卡片, 就會掉去垃圾箱, 對於紙卡用戶, 如果卡片上資料有更改，佢地無辦法即時製作名片，佢地必須重印一疊全新卡片。 當佢地唔見左卡片, 或者重印卡片時，都會產生額外嘅污染。 換句話說，適合紙卡用戶更重要, 可以係源頭減少污染問題。

「 Identifying right goals 確定product 正確嘅目標 」

跟住落嚟, 定義出實際面臨嘅問題, Cover左 2 類用戶Point Of View Statements

“作為一個經常處理成堆名片嘅紙卡用戶, 需要自動化處理, 因此在使用該應用程序時無需任何技巧和顧慮, 因為要花時間評估每一張卡片, 並清除我並不真正需要嘅那些，這很麻煩。 "

“作為一個經常與客戶面對面嘅數字卡片應用用戶，希望在與應用交互嘅過程中無需任何指令就能直觀地操作，因為他們想要避免手動輸入錯誤, 感到不安，環可以有助於消除額外工作和跟進, 呢個很重要。”

“我地點樣可以為人設計一種無壓力、可以高效率去管理卡片, 例如當前用戶（“Daniel”）, 同時可以令實體卡片用戶參與數字化卡片App, 從而達到環保目嘅？”

「 Assumption 假設陳述 」

“我們認為名片用戶嘅問題係, 佢地在收集名片時, 希望以一種易於管理嘅方式處理，雖然係工作忙碌嘅中感到麻煩，但佢地願意繼續減少對環境嘅影響。 我們將通過提高效率, 來鼓勵他們改變習慣，從而大大簡化卡片上組織嘅時間。 會將工作流程嘅任務執行時間, 從 4 分鐘到 3 分鐘，以避免在卡管理上浪費時間。”

“基於海量嘅紙卡用戶，為了抓住這個市場機會，我們需要提升整體體驗以與名片用戶保持一致。”

「 The Solution 解決方案 」

主要通過Step base流程, 更容易完成任務 <- 針對第2痛點：操作複雜

Task flow 係個App重要組成部分, 包含三個核心功能：卡片創建、卡片交換和紙卡數字化, 提供1個更視覺化嘅Step-base flow，減輕Users認知負擔, 直觀地完成任務。 專注任務, 而不是學習1個新事物, 提高兩種類型users嘅動機, 適應現實生活中場景使用。

之後, 下面第1點 降低任務複雜度，更容易數字化

一個新功能, 可以將拍攝卡片後嘅圖像, 保留為數字卡片外觀, 可以跳過設計卡片嘅過程 - 適合想要快速完成任務, 或者唔知道點樣設計卡片嘅users。 嘅可以讓紙卡用戶輕鬆地進行數字化，也可以加快任務時間。 <- 針對第2痛點：任務操作複雜,

每個Step 都有輕鬆嘅標題，例如 “選擇你喜歡嘅卡片teamplate?” 、 “我嘅信息是咩”, 標題唔係命令式, Step1 請輸入咩, Step2做呢樣, 可以輕鬆引導Users進一步嘅行動決策。 消除users壓力。

- 提供Template feature，可以加快完成, 唔需要任何設計技巧, 亦都可加快流程

- 提供雲嘅存儲 - 可以重複使用卡片template，例如有一定規模公司, 可以標準化佢地卡片，避免浪費時間重複工作。

第2點 通過 NFC 交互功能, 轉變卡片嘅整理習慣 <- 針對第1痛點：冇時間去組織卡片, 感覺額外工作

減少了額外嘅卡片組織和事後時間

構建1個NFC 交換卡片流程，即時, 同自動化嘅，避免數據太多輸入, 同埋更好嘅準確性, 透過呢個便利性, 希望可以激勵User使用呢個App, 同埋改變佢地卡片管理習慣, 減少事後嘅額外時間做管理工作, 同跟進。

另外, 新增1個Notes功能, 卡片交換後, 可以比Users set Schedules 同 Reminder, 例如Drop低開會後跟進, 或者重要事情, 唔需要額外用其他App。

第3點 1個數字“名片盒”方式 提高易用性

將Home page做成 Digital 卡片盒 嘅設計，好似現實生活中嘅卡片盒一樣。 比單純嘅聯繫人列表, 容易理解, 視覺上會更好, 另外，將關鍵功能任務放係Home度, 容易D操作同執行。 同時著重單手操作，配合現實生活上使用。

4 紙卡數字化

第4個demo展示番由紙卡轉做Digital操作.......

當Users收到紙卡片時，係Home Click “掃描” Button, 就會開左 Camera，然後，影一張實體卡片, OCR 功能 Detect到, 卡片上每個字段，然後, 從卡片中Capture數據, 然後自動Fill入去下一步填資料欄位, 姓名、 職務、 公司名稱等等信息, 同時係可編輯嘅格式，如果有任何不正確嘅資料，User可以睇番頂部Preview圖, 手動去改。

入左data後，如果Users唔需要卡片外觀，可以Click “保留相機拍攝” 選項, 保留之前影底左作為卡片外觀。 可以立即完成任務, 整個過程比較簡單快捷。

******總結 : 呢度消除都係番Pain point 2 操作複雜, 消除不必要手動工作, 亦都有彈性, 唔洗做卡片Design, 簡化過程。 對有大量卡片轉做Digital比較方便
