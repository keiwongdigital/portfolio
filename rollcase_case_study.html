roll:Case

「 Project Goals 」

上面有D引言, 落下小小, 先講Project Goals

重新設計一個數字化卡片平台，旨在以用戶為中心的方法為商人創造無縫體驗。 同時通過簡化流程降低實體卡用戶的准入門檻，從而增加潛在市場。 該項目的高度願景是無紙化，“拯救一棵樹”； “拯救地球母親”。 你知道嗎？ 全世界每年生產 1000 億張名片，其中 58% 最終被扔進垃圾箱，然後被填埋。

是我的第一個個人項目，涉及我從事的 UX 和 UI 任務，我執行了一個端到端的過程——從了解用戶的真正需求到實施它。

「 Research 」

首先要做的是通過數據了解用戶。 我開始研究以更好地理解目標。 對 13 位每天使用實體卡和數字卡應用程序的人, 進行Facebook 小組投票、whatsApp 小組調查。 每個人都有自己的挫敗感和使用方式。 以下是他們面臨的總結問題：

實體卡 users:

組織時間不夠

生態問題/成本增加

10/13 的受訪者表示仍在使用實體卡，但其中只有 3 人使用過（甚至聽說過）數字卡應用程序。

7/13 的受訪者已經做生意很長時間了，並且存儲了大量需要整理的名片。 他們感到不耐煩和額外的工作。

Digital card users:

用戶不友好的界面

沒有提供太多功能

3/13 的受訪者使用當前應用，但體驗不佳。 在可用性方面，對複雜的步驟感到沮喪，並且在執行任務時沒有指導。 此外，他們還提到了他們的任務需要額外使用應用程序。

「 Key Insight 關鍵見解」

第1點:

他們目前如何管理名片？

會後整理名片是數字或實體卡用戶的主要痛點。他們覺得工作很麻煩，事實上，沒有人有時間整理聯繫人並手動將名片詳細信息保存到手機或放入名片盒。他們總是在尋找節省時間。在應用程序的可用性方面，他們表示這個過程感覺很複雜，降低了用戶使用應用程序的動力和預期。此外，應用內功能也無法滿足用戶當前在現實生活中的需求。

第2點:

潛在市場的機會

顯然在香港不受歡迎。根據結果，今天仍有大量用戶使用實體卡。考慮到這一點，這為潛在用戶打開了數字卡應用程序的大量開發空間。

第3點:

他們不擔心環境污染的影響！

每當組織他們的名片時，就會有一些卡片最終進入垃圾箱。此外，如果某人的信息發生變化，人們無法立即製作名片，他們必須訂購一疊全新的名片。每當人們丟失卡片和卡片的生產時，也會產生額外的污染。

「 Competitive Analysis 競爭分析 」

我開始做一些研究，看看目前有什麼。 競爭對手之間的主要功能非常相似，並進一步確定了我認為 rollcase 具有的關鍵功能，以便根據這些指標評估設計決策。 我注意到的主要區別是：

乾草堆 - 優點：強烈的視覺效果/缺點：工作屏幕太多；

Biz-Card - 優點：很明顯信息/缺點：幾乎沒有操作；

名片夾 - 優點：易於訪問/缺點：視覺效果不佳；

「 IDEATION & SOLUTION 構思與解決方案 」

「 Problem Statement 」

我知道，改變一個人的習慣和受到人們的信任是我面臨的最大挑戰，尤其是長期使用實體卡的用戶。

“我如何為商務人士設計一種無壓力、快速且環保的方式將名片應用程序數字化，並讓實體卡用戶也參與數字化？”

經歷了幾輪的構思。 然後，我創建了一個親和力圖來縮小想法並確定項目的優先級。 以下是解決方案的兩個主要問題，總結了前一階段的結果：

我試圖解決的問題是什麼？ 它究竟是如何應用的？

第1點 重新構思任務流程可增強體驗

通過直觀的基於步驟的解決方案，能夠減輕用戶的認知負擔。考慮到這一點，剝離複雜的步驟並同時保持單手操作，以確保很好地適應現實生活場景。在每一步處理中都帶有輕鬆的標題，以減輕用戶的挫敗感和工作的麻煩（例如“你想如何設計卡片”、“我的信息是什麼”）。這種很好的方式降低了實體卡用戶進入數字化的門檻，從而減少了紙卡的使用。

- 注重分步互動

在三步內建立更直觀的數字卡片創建方式，能夠輕鬆引導他們進行進一步的行動決策。為了避免手動，允許用戶從地址簿、郵件和 vCard 格式導入將確保快速和易於應用。同時，提供現成的模板，無需任何設計技巧即可加快流程，幾乎完全自動化操作。 轉變整理卡片的習慣

- 轉變整理卡片的習慣

交換過程以類似於現實生活的易消化方式呈現； 1.打開你想分享的卡片； 2. 靠近啟用 NFC 的設備或（相同的應用程序）設備；這很適合在開會後隨時激勵他們改變卡片的管理方式，減少事後的精力和時間。一旦卡片交換，允許用戶為未來的事情創建工作時間表和提醒，使其易於跟進，避免額外的應用程序使用。

- 消除從實體卡到數字化的手動工作

通過應用內 OCR 掃描儀功能，提取數據後自動填充文本字段和可編輯格式。通過這樣做，為了減少用戶對手動輸入錯誤的挫敗感，另外，將拍攝卡片後的圖像保留為數字卡片外觀，如果用戶不需要，可以跳過設計卡片的過程。

第2點 數字“名片盒”增加了便利性和可訪問性

我打算和他們以前使用的組織名片一樣做同樣的邏輯。 將家庭建立為“數字卡片箱”，就像現實生活中的卡片中心概述一樣。 這是顯示關鍵元素的好方法：CARD。 此外，一個地點的所有任務都可以輕鬆訪問，這比用純粹的聯繫人列表來描述它更容易改善功能訪問和相關的前進旅程。

「 User flow 」

一旦我確定了重點範圍和目標。接下來，我勾勒出一個流程圖，幫助我提出整個旅程的想法，並引導我完成製作想法，以及我如何設計其餘的體驗。

「 ITERATION 強調任務流程和反饋 」

我回顧了必要的功能，並專注於提高任務完成率的交互。 我起草了 3 個用於快速測試的關鍵論文模型，這些模型對這個概念至關重要：#1：家庭交互； #2：創建自己的卡片； #3：掃描卡。 我與兩名參與者進行了測試，這有助於我了解用戶以及他們如何與應用程序交互：

1. Home Interaction

在這裡，我盡量避免基於現實生活場景的不適用設計，並保持單手操作流暢。他們喜歡帶有垂直滾動方法的數字卡片盒，感覺“真實”“酷”。也許，提高了使用該應用程序的用戶的動力和預期。根據參與者的觀察，為了進一步突出數字卡片發現，我刪除了中間過濾欄中圖標的描述文本，僅顯示圖標符號。

2. 創建自己的卡片處理

參與者第一次完成基本任務時，我觀察到參與者最常使用可定制選項而不是內置模板，因為新流程能夠輕鬆引導他們採取進一步行動。參與者說完成步驟有點長。因此，我將步驟從四步簡化為三步以及一些交互元素問題。

3. 掃卡處理

在掃卡處理方面，上面是另一個例子；測試顯示用戶不確定卡片鏡頭的正確部分是否被選擇並在之後進行編輯；然而，在其實施之後，用戶忽略並遺漏了其下方的聯繫人數據。在高保真版本中，相應地添加了卡片預覽。因此，如果數據提取不正確，卡片預覽可以幫助他們進行編輯。

Iterating result 迭代結果:

經過幾輪手稿 A/B 測試，以確定哪個對用戶有效。修改後的基於步驟的框架更加簡潔明了，與之交互並成功完成了任務，這使我能夠弄清楚最終的過程。觀察他們與應用程序交互的情緒和習慣是最重要的。

主頁作為“數字卡片箱”是應用程序的核心屏幕，顯示當前卡片狀態的概覽，方便訪問並進一步輕鬆組織卡片。默認情況下，最新的卡片位於列表頂部。點擊中間欄快速過濾面板允許用戶快速整理聯繫人列表，排序類型在哪裡：最近的聯繫人、收藏夾、公司、組等。點擊卡片以顯示人員的詳細信息。用戶可以直接向左滑動電話號碼圖標撥打電話。此外，如果他們有關係，用戶可以在聯繫人列表中添加人員。此功能的靈感來自草稿迭代階段。讓用戶跟進他們的個人網絡。
