「 Project Goals 」

呢個係一個卡片APP, 第1段係引言, 落下小小, 先講Project Goals

重新設計一個數字卡片平台，以用戶為中心, 創造1個更有效率既體驗。 同時通過簡化流程, 降低實體卡片用戶既進入門檻，從而增加潛在市場。

仲有1個比較高既願景, 係無紙化，全世界每年生產 1000 億張卡片，其中 58% 最終被扔進垃圾箱，然後填田區。 (inpire by news) (負責所有 UX 和 UI task)

「 Research 」

首先去了解用戶, 理解目標先。 揾左 10 位使用實體卡片, 有用過數字卡片App的人, 係Facebook 、 同埋係whatsApp group 做左快速調查。 佢地都有自己的挫敗感 同使用方式。 下面係總結左佢地既問題：

“我經常把卡片放在西裝袋裡，有時我冇時間去整理, 最終會浪費時間揾卡片”

“開會後回到辦公室時整理卡片，大約 2 分鐘入手機”

“最近有用過APP，但唔喜歡建立卡片時既步驟複雜”

“建立卡片後無法分享給我的同事””

「 Synthesising Finding 」

下圖係sort out左User problem, 歸納左3個痛點 :

第1個痛點 ：冇時間去組織卡片, 感覺額外工作

整理卡片, 同時係數字卡App或實體卡用戶既共同痛點。 佢地覺得係額外的工作。 大部分受訪者表示仍然用實體卡片, 另外, 佢地已經做生意很長時間，而且存儲了大量需要整理的卡片。

第2個痛點：操作複雜

Users對複雜步驟 感到沮喪，並且在執行任務時, 唔夠清析D指導, 亦都未必付合唔到users實際需求。

第3個痛點：功能上不足

受訪者亦都表示Share唔到做好左卡片比同事, 無法滿足他們的任務和期望。

「 Empathize User Facing 」

消化左後, 有3點見解:

第1點: 目前, 佢地整理過程中都面臨哪些問題？

事實上，Users大多數收到卡片後, 冇即時去整理, 另外, 亦都未必會花時間, 將卡片資料手動入到手機, 或者放入自已卡片盒。 亦好容易忘記或有時會丟失。 如果要同某人聯繫, 他們通常會浪費時間找卡。

第2點: 香港比較少人用

好明顯, 今日仍有大量人用紙卡片, 呢度發現到潛在用戶, 有空間開發Digital Card。 我測試市場上了App, 可用性方面過程感覺複雜, 亦增加左紙卡片User門檻, 激發唔到User使用動機。

第3點: 佢地不擔心環境污染的影響！

每次佢地整理自已既卡片時，就會有一D卡片, 最終進入垃圾箱。 做成污染, 另外, 如果某人卡片上資料有變化，佢地無辦法即時製作名片，佢地必須重印一疊全新卡片。 佢地唔見左卡片, 或者重印卡片時，都會產生額外既污染。

「 Competitive Analysis 競爭分析 」

做左一些研究，看看目前 競爭對手之間 功能，揾個市場上定位, 發現到主要區別有以下：

Haystack - 優點：有強烈視覺效果 / 缺點：工作流程太多；

Biz-Card - 優點：清析指導 / 缺點：過多的文字描述；

Business Card Holder - 優點：容易訪問 / 缺點：視覺效果不佳；

「 Problem Statement 」

最大挑戰係 改變一個人習慣, 同比到人信任, 尤其係長期使用實體卡片既User。

“點為人設計一種無壓力、可以高效去管理自己既卡片, 同時令實體卡片用戶參與數字化卡片App, 從而達到環保目的？”

「 IDEATION 構思 」

之後, 我構思左4點策略方案:

第1係 通過Step base流程, 更容易完成任務 <- 針對第2痛點：操作複雜

- 重新構思1個視覺化的Step base flow，減輕Users認知負擔, 專注任務, 而不是學習1個新事物。

- 提供Template feature，可以加快完成, 唔需要任何設計技巧

- 用輕鬆的標題去引導, 減輕User既行動決策, 例如 “你想點設計卡片?” 、 “我既信息是咩”, (而唔糸叫User Step1做呢樣, Step2做呢樣。)

第2點 消除從實體卡片到數字化的手動工作 <- 針對第2痛點：操作複雜, )

App裡面既 OCR 掃描功能，可以自動Capture 卡片Data, 自動Fit入去, 同時係可編輯格式。 目的係，可以減少用戶對手動輸入錯誤的挫敗感, 同時可以降低實體卡User進入門襤

另外，如果用戶不需要Design, 可以將拍攝卡片後既圖像, 保留為數字卡片外觀, 亦都可以提高任務成功率。

提供雲端 Template。 User可以重複使用，亦可以標準化品牌, 避免浪費時間重複工作。 <- 針對第3個痛點：功能不足

第3點 轉變整理卡片習慣, 減少了額外的卡片組織和事後時間 <- 針對第1痛點：冇時間去組織卡片, 感覺額外工作

創建NFC功能交換流程，幾乎完全自動化，透過呢個便利性, 希望可以激勵User去整理名片的習慣, 減少事後的額外工作和時間。

新增1個Notes功能, 卡片交換後, 可以比Users即時set schedules 同 Reminder, 例如Drop低開會後跟進事情, 避免用額外App。 <- 針對第3個痛點：功能不足

第4點 1個數字“名片盒”方式 提高易用性, 鼓勵User用呢個App

將Home page做成 Digital 卡片盒 既設計，好似現實生活中既卡片盒一樣。 另外，將關鍵功能任務放係Home度, 容易D操作同執行。

1 Home

第1個demo展示番Home 基本操作, 然後打電話比巨。

另外，右圖係新增1個Feature, Users可以 Add 同佢地有關係既人, 例如同間1公司, 又或者生意上有關係既。 (呢個功能係inpire ITERATION階段)

2 Own card creation

第2個demo展示番Create自己的卡片操作.......

當User click頭像，入去“我的卡片盒”，呢個係自己卡片的中心。 可以建立自己的卡片，有3種模式：

1. 重新開始 - Step by Step 去建立卡片
2. 企業 - 創建後從雲存儲訪問
3. Scan卡 in-App OCR scanner capture實體卡片數據

呢個Demo係 Start all over 操作

第 1 步 - Import contact information，Users可以係手機 Address book，email 或者 vCard format file import入去, 可以防止手動數據輸入錯誤。 另外可以加頭像。 (通過他們的手機同步，以便聯繫信息始終是最新的)

入左基本資料後, 係第2步 有2個選項, 關於卡片Visual：

- 用 template
User可以用Template，系統將加載導入的數據，有助於簡化整個過程

- 自定 Customizable
User可以使用工具、裁剪和顏色疊加 自己定義視覺元素

咁個Demo 第 3 步 係自己定義 - 見到入logo, 入background,可以再做1D調整, 剪刧, 顏色overlay, 之後可以比Users自已調整卡片佈局、 Font大細, 同埋卡上信息顯示 ON/OFF。 Users調整時, 會睇到即時變化。 (卡片設置有更靈活的選項)

另外, Enable “設置為企業模板”，Cloud base，可以同團隊共享。 Card design 同埋Layout Startard 左做1個Template, 避免浪費時間重複工作。

****** UI上既處理, 多d視覺上元素同引導操作, 介面上係直觀, 解決番操作複雜Pain point, 藉着呢個易用性, 減底左用紙卡片User門檻, 希望可以達到環保目的。

3 Exchange Parnter card

第3個demo展示番交換卡片操作.......

Home page中間有Exchange 交換Button, 向上Swipe個Button, 就出現自己個card list, User 可以揀邊張卡去交換。 然後，靠近對方既手機，就會自動完成, 過程雙方都要enable NFC功能, 2個step完成。

交換後，就跳番去Home 會顯示新卡。 長按卡片可以編輯、 刪除、 Share 可以經由Email、 WhatsApp、 Facbook 和 Skype 去Share。

Demo所示, 幾配合到User開會後情境, 可以 Drop 事情notes, 有時間選項, 同埋 Set Reminder。 可能約左下次開會跟進, 唔洗用額外 App記低。 <- 針對第3個痛點：功能不足

通過呢個NFC自動化交換過程。激勵User去整理名片的習慣, 減少事後的額外工作和時間。

4 Import physical card

第4個demo展示番由實體卡轉做Digital卡片操作.......

當Users收到紙卡片時，係Home Click “掃描” Button, 就會開左 Camera，然後，影一張實體卡片, OCR 功能可即時 Detect到, 卡片上每個字段，然後, 從卡片中Capture數據, 然後頂部顯卡片Preview。 自動Fill入去姓名、 職務、 公司名稱等信息, 同時係可編輯的格式，如果有任何不正確的資料，User可以睇番頂部Preview圖, 手動修復。

入左data後，如果Users唔需要卡片外觀，可以Click “保留相機拍攝” 選項, 保留之前影底左作為卡片外觀。 可以立即完成任務, 有助簡化整個過程。

******呢度消除都係番Pain point 2 操作複雜, 消除不必要手動工作, 亦都有彈性, 唔洗做卡片Design, 簡化過程。
